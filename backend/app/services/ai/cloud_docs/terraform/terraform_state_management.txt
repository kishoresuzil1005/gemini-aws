# Terraform State Management on AWS
Source: HashiCorp Official Documentation

## What Is Terraform State?
Terraform state is a JSON file that maps your Terraform configuration to real-world resources. It tracks metadata, resource dependencies, and the current configuration of managed infrastructure.

## Why Remote State?
Local state files are dangerous for team environments:
- Cannot be shared safely between team members
- No locking = concurrent apply = state corruption
- No backup / versioning
- Secrets stored in plaintext on disk

## Setting Up S3 Remote State

### Step 1: Create the S3 Bucket and DynamoDB Table
```hcl
# bootstrap/main.tf
resource "aws_s3_bucket" "terraform_state" {
  bucket = "my-company-terraform-state"
  lifecycle {
    prevent_destroy = true
  }
}

resource "aws_s3_bucket_versioning" "enabled" {
  bucket = aws_s3_bucket.terraform_state.id
  versioning_configuration {
    status = "Enabled"
  }
}

resource "aws_s3_bucket_server_side_encryption_configuration" "default" {
  bucket = aws_s3_bucket.terraform_state.id
  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "aws:kms"
    }
  }
}

resource "aws_s3_bucket_public_access_block" "public_access" {
  bucket                  = aws_s3_bucket.terraform_state.id
  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}

resource "aws_dynamodb_table" "terraform_locks" {
  name         = "terraform-state-locks"
  billing_mode = "PAY_PER_REQUEST"
  hash_key     = "LockID"
  attribute {
    name = "LockID"
    type = "S"
  }
}
```

### Step 2: Configure the Backend
```hcl
# terraform.tf
terraform {
  backend "s3" {
    bucket         = "my-company-terraform-state"
    key            = "prod/us-east-1/eks/terraform.tfstate"
    region         = "us-east-1"
    encrypt        = true
    dynamodb_table = "terraform-state-locks"
  }
}
```

## State File Path Conventions
Use a consistent key naming convention:
```
{account}/{region}/{environment}/{component}/terraform.tfstate

Examples:
123456789/us-east-1/prod/vpc/terraform.tfstate
123456789/us-east-1/prod/eks/terraform.tfstate
123456789/us-east-1/prod/rds/terraform.tfstate
```

## Multi-Account State Isolation
Each AWS account should have its own state bucket:
```
Account: 123456789 (Production)  → s3://prod-terraform-state/
Account: 987654321 (Development) → s3://dev-terraform-state/
Account: 111111111 (Staging)     → s3://staging-terraform-state/
```

## Sharing State Between Components
Use `terraform_remote_state` to reference outputs from another component:
```hcl
data "terraform_remote_state" "vpc" {
  backend = "s3"
  config = {
    bucket = "my-company-terraform-state"
    key    = "prod/us-east-1/vpc/terraform.tfstate"
    region = "us-east-1"
  }
}

resource "aws_eks_cluster" "main" {
  name     = "prod-cluster"
  vpc_config {
    subnet_ids = data.terraform_remote_state.vpc.outputs.private_subnet_ids
  }
}
```

## State Operations

### Listing Resources
```bash
terraform state list
```

### Moving Resources
```bash
# Rename a resource in state
terraform state mv aws_instance.old_name aws_instance.new_name

# Move resource into a module
terraform state mv aws_instance.main module.ec2.aws_instance.main
```

### Removing Resources from State
```bash
# Remove without destroying the actual resource
terraform state rm aws_instance.deprecated
```

### Importing Existing Resources
```bash
terraform import aws_vpc.main vpc-0123456789abcdef0
```

### Refreshing State
```bash
terraform refresh
```

## State Backup and Recovery
- S3 versioning provides automatic backup of every state change
- To roll back: restore a previous version from S3 console or CLI
- Always test state rollback in non-production before needing it in production

## Security for State Files
State files often contain sensitive data (passwords, keys). Protect them:
1. Enable S3 bucket encryption (KMS)
2. Enable S3 versioning for rollback
3. Block all public access on the S3 bucket
4. Use bucket policies to restrict access to specific IAM roles
5. Enable CloudTrail logging on the S3 bucket
6. Use VPC endpoints to access S3 privately

## Terraform Cloud/Enterprise State
For large teams, consider Terraform Cloud which provides:
- Managed remote state with encryption
- State locking built-in
- Run history and audit logging
- Sentinel policy enforcement
- Team collaboration features
