# AWS Secrets Manager - Secrets Rotation & Configuration Storage Security

AWS Secrets Manager helps you protect secrets needed to access your applications, services, and IT resources. The service enables you to easily rotate, manage, and retrieve database credentials, API keys, and other secrets throughout their lifecycle.

## Core Concepts

### Secrets
A JSON string structure containing sensitive parameters (e.g., username, password, host, port, apikey).

### Automatic Secret Rotation
Direct integration with AWS Lambda to automatically update passwords and credentials, such as modifying the target RDS master password and updating the active secret value simultaneously without administrative downtime.

### Versioning
Secrets Manager automatically maintains history hashes of secrets using labels like `AWSPREVIOUS` and `AWSCURRENT` to coordinate smooth application rollouts without service disruption.

## DevSecOps Security Best Practices

### 1. Block IAM Administrator access from decrypting secrets
Do not allow generic IAM administrator roles (`"Action": "*"`) to read secrets. Create resource policies requiring specific application task role access keys to decrypt sensitive payloads.

### 2. Configure Automatic Rotation with short lifespans
Set database credentials and third-party API keys to automatically rotate every 30-90 days, locking down exposure windows of leaked credentials.

### 3. Monitor Access and Track Anomalous Pulls
Set up CloudWatch Metric Filters on Secrets Manager decryption calls (`GetSecretValue`). Alert security operations immediately if database service roles make unusual numbers of retrieval calls.
