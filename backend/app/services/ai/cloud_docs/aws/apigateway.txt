# Amazon API Gateway - API Management, Deployment, and Access Auditing

Amazon API Gateway is a fully managed service that makes it easy for developers to create, publish, maintain, monitor, and secure APIs at any scale. APIs act as the "front door" for applications to access data, business logic, or functional microservices running on AWS Fargate, EC2, or Lambda.

## Core Concepts

### API Types
*   **REST APIs:** Highly customizable, feature-rich HTTP APIs supporting request/response transformations, API keys, usage contracts, and cache options.
*   **HTTP APIs:** Optimized, high-performance, lower-cost REST alternatives designed specifically for proxying requests directly to HTTP endpoints or Lambda functions.
*   **WebSocket APIs:** Enable stateful, full-duplex persistent two-way communication between serverless backend scripts and connecting web clients.

### Integrations and Deployments
*   **Lambda Proxy Integration:** Directly forwards the raw HTTP request object to a backend Lambda function.
*   **API Gateway Stages:** Logical deployments of your API structure (e.g., `dev`, `staging`, `prod`) linked to specific configuration models.

## Security Practices and Hardening

### 1. Robust API Authentication & Authorization
Never expose APIs to the public without authorization layers unless explicitly required. Protect your endpoints using:
*   **IAM Authorizers:** Verify signatures generated by AWS clients.
*   **Cognito User Pools:** Directly validate JWT tokens originating from active mobile/web sessions.
*   **Lambda Custom Authorizers:** Code specialized token parsing or policy lookup rules.

### 2. Enable AWS WAF (Web Application Firewall)
Integrate your API Gateway stages directly with AWS WAF. Configure firewall rules to block common application exploits (such as SQL injection or Cross-Site Scripting), audit rate limit parameters to prevent denial-of-service (DDoS) traffic, and filter requests originating from unauthorized countries or IP ranges.

### 3. Implement Strict Throttling and Usage Plans
Set up default and route-level rate limits (tokens per second) to prevent malicious or misconfigured scripts from overwhelming downstream microservices, ensuring high-availability stability.
